Overview:
This guide covers common authentication failures related to passwords, account state, and domain sync delays.
Symptoms:
- “Incorrect username or password”
- “Your account has been locked”
- Repeated MFA prompts during login
- Login works on VPN but not Outlook
Troubleshooting Steps:
1. Confirm the user is entering the correct username (UPN format):
   username@company.internal
2. Check for account lockout:
   - ADUC → Right-click user → Properties → Account → Unlock Account
3. Reset password using approved process:
   - https://passwordreset.company.internal
4. Verify directory sync:
   - Changes may take up to 3 minutes to sync from Azure AD to on-prem AD.
5. Clear cached credentials:
   - Windows: Credential Manager → Windows Credentials → Remove stale entries.
   - macOS: Keychain Access → search for “company” → delete old items.
6. Check device time:
   - Incorrect system time causes token/MFA validation failures.
Escalation:
If authentication still fails, gather:
- Screenshot of error
- Timestamp of failure
- Device used
Then escalate to IAM Team.
